<objective>
Wire SunoClient into study_with_me_generator.py so Suno task submission happens in a
background thread before the SDXL image batch starts, and the result is collected after
images complete — hiding Suno's 60-180s latency behind SDXL's generation time.

Purpose: SUNO-07 requires that Suno submission precedes SDXL so the user experiences
near-zero added wait time for music. This plan adds the threading wiring and progress
tracking while preserving all existing orchestrator behavior when Suno is not configured.

Output: study_with_me_generator.py updated with ThreadPoolExecutor wiring; no new files.
</objective>
