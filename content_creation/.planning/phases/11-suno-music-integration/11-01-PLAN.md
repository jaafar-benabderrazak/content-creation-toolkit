Create generators/suno.py with the following structure. Do NOT import pipeline_config
directly inside the class ŌĆö SunoClient receives SunoSettings as a typed argument to
keep the boundary clean (same pattern as SDXLGenerator receiving SDXLSettings).

**Module constants:**
```python
SUNO_BASE_URL = os.environ.get("SUNO_BASE_URL", "https://api.sunoapi.org")
SUNO_MAX_TRACK_SECONDS = 240   # conservative; actual Suno v5 max ~4 min
POLL_INITIAL_WAIT = 5          # seconds
POLL_MAX_WAIT = 60             # cap for exponential backoff
POLL_HARD_TIMEOUT = 300        # seconds ŌĆö total wall-clock limit
```

**SunoClient class:**

`__init__(self, suno_cfg: SunoSettings)` ŌĆö store cfg, build requests.Session with
`Authorization: Bearer {suno_cfg.api_key}` and `Content-Type: application/json` headers.
Raise ValueError if api_key is None (SUNO_API_KEY env var not set).

`generate_music(self, prompt: str, duration_seconds: int, genre: str) -> AudioSegment`
ŌĆö primary interface, must return pydub.AudioSegment, never raise.
Logic:
1. If duration_seconds > SUNO_MAX_TRACK_SECONDS: compute n_tracks = ceil(duration / SUNO_MAX_TRACK_SECONDS); submit a generation request for each chunk, collect all task_ids.
2. Else: submit one request for `duration_seconds`; single task_id.
3. For each task_id: call `_poll_until_complete(task_id)` ŌåÆ returns audio_url string.
4. Download each audio_url to a temp file via `_download_audio(url)` ŌåÆ Path.
5. Convert each to AudioSegment via `AudioSegment.from_file(path)`.
6. Call `_stitch_tracks(segments, target_duration_ms)` ŌåÆ single AudioSegment trimmed to exact length.
7. Return the stitched AudioSegment.
Wrap the entire body in try/except Exception ŌĆö on any failure call `_stable_audio_fallback(prompt, duration_seconds)` and return its result. Log the failure trigger with `logging.warning("[Suno] Fallback triggered: %s", e)`.

`_submit(self, prompt: str, duration: int, genre: str) -> str`
ŌĆö POST to `{SUNO_BASE_URL}/api/v1/generate` with payload:
```json
{
  "prompt": "<genre> <prompt>",
  "make_instrumental": true,
  "duration": <duration>,
  "mv": "<self.suno_cfg.model_version>",
  "count": <self.suno_cfg.track_count>
}
```
Returns task_id from `response.json()["task_id"]`. Raise `requests.HTTPError` on non-2xx.

`_poll_until_complete(self, task_id: str) -> list[str]`
ŌĆö Poll GET `{SUNO_BASE_URL}/api/v1/task/{task_id}` with exponential backoff.
Use manual timing loop (NOT tenacity decorator ŌĆö we need to log elapsed time):
```python
wait = POLL_INITIAL_WAIT
elapsed = 0
while elapsed < POLL_HARD_TIMEOUT:
    resp = self._session.get(...)
    data = resp.json()
    status = data.get("status", "")
    if status == "completed":
        # extract audio_urls from data["data"] list ŌĆö list of track objects
        return [track["audio_url"] for track in data.get("data", [])]
    if status == "error":
        raise RuntimeError(f"Suno task failed: {data}")
    logging.info("[Suno] Polling task %s... status=%s elapsed=%ds", task_id[:8], status, elapsed)
    time.sleep(wait)
    elapsed += wait
    wait = min(wait * 2, POLL_MAX_WAIT)
raise TimeoutError(f"Suno task {task_id} exceeded {POLL_HARD_TIMEOUT}s timeout")
```

`_download_audio(self, url: str) -> Path`
ŌĆö Stream GET url to a named temp file in /tmp. Return Path to the downloaded file.
Use `tempfile.NamedTemporaryFile(suffix=".mp3", delete=False)`.

`_stitch_tracks(self, segments: list[AudioSegment], target_ms: int) -> AudioSegment`
ŌĆö Concatenate all segments with 3000ms crossfade using pydub `.append()`.
Trim to target_ms. Return result. Handles single-segment case (no stitch needed).

`_stable_audio_fallback(self, prompt: str, duration_seconds: int) -> AudioSegment`
ŌĆö Reproduce the minimal working core of `generate_enhanced_music` from
study_with_me_generator.py: import stable_audio_tools lazily, load model, generate
one or more segments until duration is covered, crossfade-join, return AudioSegment.
If stable_audio_tools is not installed: return AudioSegment.silent(duration=duration_seconds * 1000).

**Imports at module top level (non-GPU only):**
`import os, logging, time, tempfile, math` and `import requests` (not lazy ŌĆö always needed).
`from pydub import AudioSegment` ŌĆö not lazy (contract requirement).
`from config.pipeline_config import SunoSettings` ŌĆö for type hint only.
Defer `torch`, `torchaudio`, `stable_audio_tools` imports inside `_stable_audio_fallback`.

Note on field names: sunoapi.org uses `task_id` in submit response and `data[].audio_url`
in status response. If a live test shows different field names, adapt but add a comment:
`# Field name per sunoapi.org docs ŌĆö may differ on kie.ai`.
