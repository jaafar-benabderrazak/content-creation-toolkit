  <action>
Add 4 new required keys to the PromptGenerator JSON contract:

1. Update `_REQUIRED_KEYS` — add: `"thumbnail_text"`, `"youtube_title"`, `"youtube_description"`, `"youtube_tags"`

2. Update `_USER_PROMPT` — append these lines to the instruction block (keep existing 4 keys, add 4 new lines after `music_prompt`):
   ```
   - thumbnail_text: short punchy text overlay for the thumbnail (3-7 words, ALL CAPS, derived from the positive prompt atmosphere — e.g. "MIDNIGHT STUDY VIBES" or "GOLDEN HOUR FOCUS")
   - youtube_title: SEO-optimized YouTube title (60 chars max) including mood/atmosphere keywords derived from the tags and positive_prompt style
   - youtube_description: 150-200 word YouTube description with relevant keywords derived from scene themes; include 2-3 paragraph breaks; end with a call to subscribe
   - youtube_tags: JSON array of exactly 15-20 SEO tags derived from the tags input and generated content; mix broad (lofi study, study with me) and specific (rainy day studying, cozy study session)
   ```

3. Update `_validate()` — the existing check `missing = _REQUIRED_KEYS - data.keys()` already covers new keys because `_REQUIRED_KEYS` is updated. Add one more check: verify `youtube_tags` is a list with 15-20 items:
   ```python
   ytags = data.get("youtube_tags", [])
   if not isinstance(ytags, list):
       raise PromptGenerationError(
           f"youtube_tags must be a list, got {type(ytags).__name__}"
       )
   if not (15 <= len(ytags) <= 20):
       raise PromptGenerationError(
           f"youtube_tags must have 15-20 items, got {len(ytags)}"
       )
   ```

Do NOT change `_generate_claude` or `_generate_openai` — they pass the prompts through unchanged; the new fields are purely driven by the updated `_USER_PROMPT` and `_REQUIRED_KEYS`.

Note: `response_format={"type": "json_object"}` in the OpenAI path already handles free-form JSON; Claude already returns JSON per the system prompt. No provider-side changes needed.
