  <name>Task 1: Install supabase-py and write migration script</name>
  <files>scripts/migrate_roadmap.py</files>
  <action>
    Install the Supabase Python client:
    ```
    pip install supabase
    ```

    Create `scripts/migrate_roadmap.py`. The script must:

    1. Load SUPABASE_URL and SUPABASE_ANON_KEY from environment variables (or accept as CLI args with argparse). Read them from the 25-01-SUMMARY.md if available.
    2. Read `video_roadmap.json` from the repo root (relative path, since script is run from repo root).
    3. Parse the JSON array into a list of dicts — each dict corresponds to one row in the `videos` table.
    4. Add a `position` field to each row equal to its 0-based index in the array (preserves original list order).
    5. Convert `created_at` and `updated_at` ISO strings — they are naive datetimes from Python's `datetime.utcnow().isoformat()` — append "Z" suffix if no timezone info present to make them valid TIMESTAMPTZ.
    6. Batch-upsert in chunks of 50 using `supabase.table("videos").upsert(chunk, on_conflict="id").execute()`.
    7. After all batches, run a count query and print the result.
    8. Exit with code 0 on success, 1 on any error.

    Script structure:
    ```python
    import json, os, sys
    from pathlib import Path
    from supabase import create_client

    SUPABASE_URL = os.environ["SUPABASE_URL"]
    SUPABASE_KEY = os.environ["SUPABASE_ANON_KEY"]

    def fix_timestamp(ts: str) -> str:
        if ts and not ts.endswith("Z") and "+" not in ts:
            return ts + "Z"
        return ts

    def main():
        data = json.loads(Path("video_roadmap.json").read_text(encoding="utf-8"))
        rows = []
        for i, entry in enumerate(data):
            rows.append({
                "id": entry["id"],
                "title": entry["title"],
                "tags": entry.get("tags", ""),
                "profile": entry.get("profile", "lofi-study"),
                "notes": entry.get("notes", ""),
                "status": entry.get("status", "planned"),
                "position": i,
                "created_at": fix_timestamp(entry.get("created_at", "")),
                "updated_at": fix_timestamp(entry.get("updated_at", "")),
            })

        client = create_client(SUPABASE_URL, SUPABASE_KEY)
        chunk_size = 50
        for start in range(0, len(rows), chunk_size):
            chunk = rows[start:start + chunk_size]
            result = client.table("videos").upsert(chunk, on_conflict="id").execute()
            print(f"Upserted rows {start}–{start + len(chunk) - 1}")

        count_result = client.table("videos").select("id", count="exact").execute()
        print(f"Total rows in videos table: {count_result.count}")
        assert count_result.count >= len(rows), f"Expected {len(rows)}, got {count_result.count}"
        print("Migration complete.")

    if __name__ == "__main__":
        main()
    ```

    Do NOT use the Supabase MCP tools for the migration itself — the Python script is the artifact. The MCP tools are only used to verify the result.
