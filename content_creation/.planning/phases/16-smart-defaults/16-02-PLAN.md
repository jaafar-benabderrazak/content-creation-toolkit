**dashboard/src/app/api/config/profiles/[name]/route.ts** — update the GET handler:

The route currently reads YAML from disk and returns the raw parsed object. Change it to:

1. After parsing YAML into `data`, compute provenance by checking each known env var against the corresponding field in data. Use an inline map (mirror of Python's ENV_VAR_MAP):
   ```typescript
   const ENV_VAR_MAP: Record<string, string> = {
     "notify.discord_webhook_url": "NOTF_DISCORD_WEBHOOK_URL",
     "notify.slack_webhook_url":   "NOTF_SLACK_WEBHOOK_URL",
     "suno.api_key":               "SUNO_API_KEY",
   };
   ```

2. Build provenance:
   ```typescript
   const provenance: Record<string, "env" | "yaml"> = {};
   for (const [fieldPath, envVar] of Object.entries(ENV_VAR_MAP)) {
     const [section, key] = fieldPath.split(".");
     const yamlValue = (data as Record<string, Record<string, unknown>>)[section]?.[key];
     const envValue = process.env[envVar];
     if ((yamlValue === null || yamlValue === undefined || yamlValue === "") && envValue) {
       provenance[fieldPath] = "env";
     } else {
       provenance[fieldPath] = "yaml";
     }
   }
   ```

3. Return `{ config: data, provenance }` instead of `data` directly.

Note: `process.env` is available in Next.js API routes. Do NOT use `NEXT_PUBLIC_` prefix — these are server-side secrets. The env vars are read server-side only; they never reach the browser except as the provenance string "env"/"yaml".
