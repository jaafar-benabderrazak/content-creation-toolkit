  <action>
Create `setup.py` at project root. This is a standalone CLI script — not a distutils/setuptools setup.py. Add a shebang comment clarifying this:

```python
#!/usr/bin/env python
"""
First-run setup: scans environment variables and writes a starter pipeline profile.

Usage:
    python setup.py [--output config/profiles/starter.yaml] [--force]

Options:
    --output PATH   Output YAML path (default: config/profiles/starter.yaml)
    --force         Overwrite existing file without prompting
"""
```

Implementation:

1. Define `KNOWN_ENV_VARS` — an ordered list of dicts describing every env var the pipeline recognises:
   ```python
   KNOWN_ENV_VARS = [
       {"env": "NOTF_DISCORD_WEBHOOK_URL", "field": "notify.discord_webhook_url", "description": "Discord webhook for pipeline notifications"},
       {"env": "NOTF_SLACK_WEBHOOK_URL",   "field": "notify.slack_webhook_url",   "description": "Slack webhook for pipeline notifications"},
       {"env": "SUNO_API_KEY",             "field": "suno.api_key",               "description": "Suno music generation API key (kie.ai)"},
       {"env": "OPENAI_API_KEY",           "field": None,                         "description": "OpenAI API key (used in Phase 18 prompt generation)"},
       {"env": "REPLICATE_API_TOKEN",      "field": None,                         "description": "Replicate API token for SDXL fallback"},
       {"env": "YOUTUBE_QUOTA_USED",       "field": None,                         "description": "YouTube daily quota tracker (set by publisher after upload)"},
   ]
   ```
   Fields with `field: None` are noted but not injected into PipelineConfig (no corresponding model field exists yet).

2. Parse args with `argparse`:
   - `--output` default `"config/profiles/starter.yaml"`
   - `--force` flag (store_true)

3. Scan env vars:
   ```python
   found = {}   # env_var_name -> value
   missing = [] # env_var_name
   for entry in KNOWN_ENV_VARS:
       val = os.environ.get(entry["env"])
       if val:
           found[entry["env"]] = val
       else:
           missing.append(entry["env"])
   ```

4. Build a PipelineConfig with defaults, then inject found values for mapped fields:
   - Start with `config = PipelineConfig()` (all defaults)
   - For each entry in KNOWN_ENV_VARS where `entry["field"]` is not None and env var was found:
     - Parse the dotted path (`notify.discord_webhook_url` -> section="notify", key="discord_webhook_url")
     - Use `model_copy(update=...)` to apply:
       ```python
       section_obj = getattr(config, section)
       updated_section = section_obj.model_copy(update={key: found[env_var]})
       config = config.model_copy(update={section: updated_section})
       ```
     - Add `try/except AttributeError` fallback using `.copy(update=...)` for Pydantic v1

5. Check if output file already exists:
   - If yes and `--force` not set: print a warning and exit without writing (tell user to use --force)

6. Write the profile:
   ```python
   output_path = Path(args.output)
   output_path.parent.mkdir(parents=True, exist_ok=True)
   config.to_yaml(output_path)
   ```

7. Print summary to stdout:
   ```
   Starter profile written to: config/profiles/starter.yaml

   Found (pre-filled):
     NOTF_DISCORD_WEBHOOK_URL  -> notify.discord_webhook_url
     SUNO_API_KEY              -> suno.api_key

   Missing (set these in .env or export before running pipelines):
     NOTF_SLACK_WEBHOOK_URL    — Slack webhook for pipeline notifications
     OPENAI_API_KEY            — OpenAI API key (used in Phase 18 prompt generation)
     REPLICATE_API_TOKEN       — Replicate API token for SDXL fallback
     YOUTUBE_QUOTA_USED        — YouTube daily quota tracker (set by publisher after upload)
   ```

8. Exit code 0 always (missing vars are informational, not errors).

Guard the main block with `if __name__ == "__main__":`.
