All four routes follow the same response shape:
```ts
type CreditResponse = {
  provider: string;      // "suno" | "replicate" | "openai" | "youtube"
  label: string;         // Human-readable balance label e.g. "Credits Remaining"
  value: number | null;  // null = could not fetch or not configured
  unit: string;          // "credits" | "USD" | "units"
  configured: boolean;   // false if the required env var is missing
  error?: string;        // present on fetch failure
  topup_url: string;     // billing page URL for this provider
}
```

**`dashboard/src/app/api/credits/suno/route.ts`**:
- Check `process.env.SUNO_API_KEY` — if missing, return `{ provider: "suno", configured: false, value: null, ... }`
- Call kie.ai credits endpoint: GET `https://api.kie.ai/api/v1/account/credits` with header `Authorization: Bearer ${SUNO_API_KEY}`
- If response has `data.credits` field, return it as `value`
- If the endpoint does not exist or returns unexpected shape, fall back to: GET `https://api.kie.ai/api/v1/account` and look for credits/balance field
- On HTTP error: return `{ configured: true, value: null, error: "HTTP {status}" }`
- `topup_url`: `"https://kie.ai/pricing"`

Note: kie.ai API shape is not officially documented. If the credits endpoint 404s, log the response body and return `{ error: "credits endpoint not available — check kie.ai dashboard manually", value: null }`. Do not throw — always return a CreditResponse.

**`dashboard/src/app/api/credits/replicate/route.ts`**:
- Check `process.env.REPLICATE_API_TOKEN`
- Call: GET `https://api.replicate.com/v1/account` with `Authorization: Token ${REPLICATE_API_TOKEN}`
- The account endpoint returns `{ username, type, github_url }` — Replicate does NOT expose credit balance via API as of 2026
- Return `{ provider: "replicate", configured: true, value: null, label: "See dashboard", unit: "USD", error: "Balance not available via API", topup_url: "https://replicate.com/account/billing" }`
- Only check that the token is valid (200 from /account) — if 401 return configured: false

**`dashboard/src/app/api/credits/openai/route.ts`**:
- Check `process.env.OPENAI_API_KEY`
- OpenAI exposes credit balance via: GET `https://api.openai.com/dashboard/billing/credit_grants` with `Authorization: Bearer ${OPENAI_API_KEY}`
- Response has `total_available` (in USD cents or dollars — verify at runtime)
- Return `value: data.total_available`, `unit: "USD"`
- If endpoint returns 404 (usage-based orgs don't have grants): return `{ value: null, error: "Usage-based account — check dashboard", topup_url: "https://platform.openai.com/account/billing/overview" }`
- `topup_url`: `"https://platform.openai.com/account/billing/overview"`

**`dashboard/src/app/api/credits/youtube/route.ts`**:
- Do NOT call YouTube API — quota cannot be checked without a user auth context (service account not applicable here)
- Read from env vars set by the local pipeline: `YOUTUBE_QUOTA_USED` and `YOUTUBE_QUOTA_DATE`
- Compute: `remaining = 10000 - parseInt(process.env.YOUTUBE_QUOTA_USED ?? "0")`
- Return: `{ provider: "youtube", value: remaining, unit: "units", label: "Daily Quota Remaining", configured: true, topup_url: "https://console.cloud.google.com/iam-admin/quotas" }`
- Note: These env vars must be updated by the local pipeline via `vercel env add` or via Vercel dashboard after each run. Add a comment explaining this constraint.

All routes: use `export const runtime = 'nodejs'` (not edge) since we use fetch with arbitrary external URLs.
Set `export const revalidate = 0` to disable caching — credits must be fresh on every request.

Render using shadcn `<Card>`:
```

  <CardHeader className="flex flex-row items-center justify-between">
    <CardTitle>{provider} — {label}</CardTitle>
    {!configured && <Badge variant="secondary">Not configured</Badge>}
  </CardHeader>
  <CardContent>
    {loading && <Skeleton className="h-8 w-24" />}

    </Button>
  </CardFooter>
</Card>
```

**`dashboard/src/app/credits/page.tsx`** — `"use client"` page:

- State: `credits: Record<string, CreditResponse | null>` — keyed by provider
- State: `loading: boolean`

- `fetchAll`:
  ```ts
  const providers = ["suno", "replicate", "openai", "youtube"];
  const results = await Promise.allSettled(providers.map(p => fetch(`/api/credits/${p}`).then(r => r.json())));
  ```
  Update state for each settled result.
- Layout: heading "Credit & Quota Monitor", a Refresh button (calls fetchAll, sets loading=true during fetch), then a responsive grid of 4 `<CreditCard>` components: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4`
- Pass `loading={loading}` to each card while fetching
