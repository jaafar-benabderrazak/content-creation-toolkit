Create `dashboard/vercel.json`:
```json
{
  "framework": "nextjs",
  "buildCommand": "npm run build",
  "outputDirectory": ".next",
  "installCommand": "npm install",
  "regions": ["cdg1"]
}
```
`cdg1` = Paris (nearest to the user's location based on Windows locale context). Adjust if deploying to a different region.

Create `dashboard/.env.example` documenting every environment variable the dashboard will need (all plans):
```
# Suno (kie.ai) — used by /api/credits/suno
SUNO_API_KEY=your_kie_ai_api_key_here

# Replicate — used by /api/credits/replicate
REPLICATE_API_TOKEN=your_replicate_token_here

# OpenAI — used by /api/credits/openai
OPENAI_API_KEY=sk-...

# Pipeline webhook secret — used by /api/status to authenticate incoming POST from local pipeline
PIPELINE_WEBHOOK_SECRET=generate_a_random_32_char_string_here

# Pipeline trigger webhook — the local machine must expose this URL (via ngrok or similar)
# The dashboard calls this URL to trigger pipeline execution
PIPELINE_TRIGGER_URL=https://your-ngrok-url.ngrok.io/trigger

# YouTube quota (optional — read from YOUTUBE_QUOTA_USED env var set by the local pipeline before push)
YOUTUBE_QUOTA_USED=0
YOUTUBE_QUOTA_DATE=2026-01-01
```

Create `dashboard/.gitignore`:
```
.env.local
.env
.next/
node_modules/
*.log
```

No `.env.local` should be committed. Document in the .env.example that users must create `.env.local` from `.env.example`.

Note on architecture: The local Python pipeline will POST status updates to the Vercel deployment using `PIPELINE_WEBHOOK_SECRET` for auth. The dashboard triggers the pipeline by calling `PIPELINE_TRIGGER_URL` — a local webhook (e.g., ngrok-exposed Flask endpoint). This is the correct pattern for a Vercel remote control panel over a local GPU process.
