**`dashboard/src/app/api/config/profiles/route.ts`** — GET handler:
- Resolve profiles directory: `path.resolve(process.cwd(), '../config/profiles')`
- Read all `.yaml` files with `fs.readdirSync`
- Return `{ profiles: string[] }` — just the filenames without extension (e.g., `["cinematic", "lofi_study", "tech_tutorial"]`)
- On error (directory missing) return 500 with `{ error: "profiles directory not found" }`

**`dashboard/src/app/api/config/profiles/[name]/route.ts`** — GET and PUT:

GET:
- Validate name matches `/^[a-z0-9_-]+$/` — return 400 if not
- Build path: `path.resolve(process.cwd(), '../config/profiles', name + '.yaml')`
- If file does not exist: return 404 `{ error: "Profile not found: {name}" }`
- Read file, parse with `yaml.load()`, return as JSON with `Content-Type: application/json`

PUT:
- Parse request body as JSON
- Validate name regex same as GET
- Build path same as GET
- Convert body back to YAML string with `yaml.dump(body, { lineWidth: 120 })`
- Write to file with `fs.writeFileSync`
- Return 200 `{ saved: true, profile: name }`
- On any error return 500 with `{ error: string }`

Important: The API routes use `path.resolve(process.cwd(), '../config/profiles')` because `process.cwd()` in Next.js is the `dashboard/` directory, and the profiles live one level up at `config/profiles/`. This works in local dev. For Vercel deployment, the profiles path won't exist on the server — but the dashboard is intended for local dev and preview deployments where the git checkout is present. Add a comment noting this constraint.

Do NOT use `fs` in client components — all file I/O is server-side only inside these route handlers.

```ts
export interface VideoSettings {
  style_prompt: string;
  music_prompt: string;
  mood: string;

  scene_duration_seconds: number;
  enable_weather: boolean;
  enable_time_progression: boolean;
  enable_parallax: boolean;
  enable_particles: boolean;
  parallax_strength: number;
}
export interface PostSettings { watermark_enabled: boolean; watermark_text: string; watermark_position: string; subtitles_enabled: boolean; subtitles_srt_path: string; intro_enabled: boolean; intro_clip_path: string; outro_enabled: boolean; outro_clip_path: string; }
export interface PublishSettings { youtube_enabled: boolean; youtube_title: string; youtube_description: string; youtube_tags: string[]; youtube_category_id: string; youtube_privacy: "public" | "unlisted" | "private"; thumbnail_enabled: boolean; }
export interface NotifySettings { discord_webhook_url: string | null; slack_webhook_url: string | null; require_approval: boolean; approval_timeout_seconds: number; }
export interface PipelineConfig { profile: string; video: VideoSettings; post: PostSettings; publish: PublishSettings; notify: NotifySettings; sdxl?: Record<string, unknown>; suno?: Record<string, unknown>; }
```
