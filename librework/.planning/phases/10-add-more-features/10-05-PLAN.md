    1. Run `cd librework/frontend && npm install next-intl`.

    2. Create `src/i18n/routing.ts`:
       ```
       import { defineRouting } from 'next-intl/routing'
       export const routing = defineRouting({
         locales: ['en', 'fr'],
         defaultLocale: 'en',
       })
       ```

    3. Create `src/i18n/request.ts`:
       ```
       import { getRequestConfig } from 'next-intl/server'
       import { routing } from './routing'
       export default getRequestConfig(async ({ requestLocale }) => {
         let locale = await requestLocale
         if (!locale || !routing.locales.includes(locale as any)) {
           locale = routing.defaultLocale
         }
         return {
           locale,
           messages: (await import(`../../messages/${locale}.json`)).default,
         }
       })
       ```

    4. Create `src/middleware.ts` (at src root, NOT inside app/):
       - Import `createMiddleware` from 'next-intl/middleware'.
       - Import `routing` from './i18n/routing'.
       - Export default `createMiddleware(routing)`.
       - Export `const config = { matcher: '/((?!api|handler|_next|_vercel|.*\\..*).*)', }`.
       - CRITICAL: The matcher MUST exclude `api` (Stripe webhook), `handler` (Stack Auth), `_next`, `_vercel`, and static files. The negative lookahead `(?!api|handler|...)` ensures these paths are never locale-prefixed.

    5. Create `messages/en.json` with all user-facing strings organized by component namespace:
       - `Navbar`: explore, myBookings, login, register, logout, ownerDashboard, languageSwitcher
       - `HomePage`: title, subtitle, searchPlaceholder, ctaButton
       - `ExplorePage`: title, filters, noResults, mapView, listView
       - `UserDashboard`: title, reservations, paymentHistory, noReservations, pay, cancel
       - `OwnerDashboard`: title, revenue, occupancy, bookings, analytics
       - `Auth`: loginTitle, registerTitle, email, password, submit, forgotPassword
       - `Common`: loading, error, retry, save, delete, confirm, back

    6. Create `messages/fr.json` with French translations for ALL keys in en.json. Use natural French (not machine-translated).

    7. Update `next.config.js`: Add `const createNextIntlPlugin = require('next-intl/plugin'); const withNextIntl = createNextIntlPlugin('./src/i18n/request.ts');` and wrap the existing config export: `module.exports = withNextIntl(existingConfig)`.

    CRITICAL PITFALL: Middleware file goes at `src/middleware.ts` (Next.js picks it up from src/ when using src directory).
