  <action>
    1. Add `stripe` to `backend/requirements.txt` (pip install stripe).

    2. Add to `config.py` Settings class:
       - `STRIPE_SECRET_KEY: str = ""`
       - `STRIPE_WEBHOOK_SECRET: str = ""`
       - `FRONTEND_URL: str = "http://localhost:3000"`

    3. Create `backend/app/api/v1/payments.py` with:
       - `router = APIRouter(prefix="/payments", tags=["Payments"])`
       - `POST /checkout-session`: Accepts `reservation_id` (str). Uses `Depends(get_current_user)`. Fetches reservation from Supabase `reservations` table joined with `spaces(name)` to get `cost_credits` and space name. Validates reservation belongs to current user and status is "pending". Computes `amount_cents = cost_credits * 100` (1 credit = 1 EUR cent). Creates `stripe.checkout.Session` with `payment_method_types=["card"]`, `mode="payment"`, `line_items` with price_data (currency "eur", product_data name from space, unit_amount from DB), `success_url` pointing to `{FRONTEND_URL}/booking/success?session_id={CHECKOUT_SESSION_ID}`, `cancel_url` to `{FRONTEND_URL}/booking/cancel`, and `metadata={"reservation_id": reservation_id, "user_id": current_user.id}`, `customer_email=current_user.email`. Returns `{"checkout_url": session.url}`.
       - `PATCH /reservations/{reservation_id}/confirm`: Internal endpoint for webhook callback. Accepts `X-Webhook-Secret` header, validates it matches `STRIPE_WEBHOOK_SECRET`. Updates reservation status to "confirmed" in Supabase. Returns 200. This avoids needing the webhook to have Supabase credentials directly.
       - `GET /payments/history`: Returns payment history for current user. Queries reservations with status "confirmed" or "completed", including `stripe_session_id` field if stored. For v1, returns reservation data as payment records (no separate payments table needed).

    4. Register payments router in `main.py`: `app.include_router(payments.router, prefix=f"{settings.API_V1_PREFIX}")`.

    IMPORTANT: Never accept price/amount from the request body. Always fetch from DB. Use `stripe.api_key = settings.STRIPE_SECRET_KEY` at module level.

    1. Run `cd librework/frontend && npm install stripe @stripe/stripe-js`.

    2. Create `src/app/api/stripe/webhook/route.ts`:
       - Import `Stripe` from 'stripe'.
       - Instantiate `const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!)`.
       - Export `POST` handler: read raw body via `const body = await req.text()` (NOT req.json() — required for signature verification). Get `stripe-signature` header. Call `stripe.webhooks.constructEvent(body, sig, process.env.STRIPE_WEBHOOK_SECRET!)` in try/catch. On `checkout.session.completed`, extract `reservation_id` from `session.metadata`. Call FastAPI internal endpoint `PATCH /api/v1/payments/reservations/{reservationId}/confirm` with `X-Webhook-Secret` header set to `process.env.STRIPE_WEBHOOK_SECRET`. Use `process.env.INTERNAL_API_URL || process.env.NEXT_PUBLIC_API_URL` as base URL. Return 200 on success, 400 on signature failure.

    3. Create `src/app/api/stripe/checkout/route.ts`:
       - Export `POST` handler: proxy request to FastAPI `POST /api/v1/payments/checkout-session`. Forward the Authorization header from the incoming request. Return the `checkout_url` from FastAPI response. This keeps the Stripe secret key on the server side and lets the frontend call a same-origin route.

    CRITICAL: The webhook route MUST use `req.text()` not `req.json()`. The middleware matcher (when i18n is added later) must exclude `/api` paths — but that is Plan 05's concern, not this plan's.
