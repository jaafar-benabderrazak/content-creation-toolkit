  <action>
    **Part A: Stack Auth server config (frontend)**

    Create NEW file `frontend/src/stack/server.ts`:
    ```typescript
    import { StackServerApp } from "@stackframe/stack";

    export const stackServerApp = new StackServerApp({
      projectId: process.env.NEXT_PUBLIC_STACK_PROJECT_ID!,
      publishableClientKey: process.env.NEXT_PUBLIC_STACK_PUBLISHABLE_CLIENT_KEY!,
      secretServerKey: process.env.STACK_SECRET_SERVER_KEY!,
      urls: {
        afterSignIn: "/",
        afterSignUp: "/",
        afterSignOut: "/handler/sign-in",
      },
    });
    ```
    Note: `STACK_SECRET_SERVER_KEY` is NOT prefixed with NEXT_PUBLIC_ — it's server-only.

    **Part B: Owner self-promotion API route (frontend)**

    Create NEW file `frontend/src/app/api/become-owner/route.ts`:
    - Import `stackServerApp` from `@/stack/server`
    - Export async `POST` handler:
      1. Get the current user from Stack Auth: `const user = await stackServerApp.getUser()` — if no user, return 401
      2. Check current role: `const currentRole = user.clientReadOnlyMetadata?.role || "customer"`. If already "owner" or "admin", return 200 with message "Already an owner/admin"
      3. Update Stack Auth metadata: `await user.update({ clientReadOnlyMetadata: { ...user.clientReadOnlyMetadata, role: "owner" } })`
      4. Also update local Supabase users table directly (don't wait for webhook): call Supabase to update `role = 'owner'` where `stack_auth_id = user.id`. Use the Supabase client from the backend or a server-side Supabase client.
      5. Return 200 `{ success: true, role: "owner" }`

    IMPORTANT per user decision: Self-service, no admin approval. The route just updates the role immediately.

    For the Supabase update in step 4, use the Supabase JS client with the service role key (server-side only):
    ```typescript
    import { createClient } from "@supabase/supabase-js";
    const supabase = createClient(
      process.env.NEXT_PUBLIC_SUPABASE_URL!,
      process.env.SUPABASE_SERVICE_ROLE_KEY!
    );
    await supabase.from("users").update({ role: "owner" }).eq("stack_auth_id", user.id);
    ```
    Add `SUPABASE_SERVICE_ROLE_KEY` to .env.local.example if not already there.

    **Part C: Webhook receiver (backend)**

    Create NEW file `backend/app/api/v1/webhooks.py`:
    - Create `router = APIRouter(prefix="/webhooks", tags=["Webhooks"])`
    - Create `POST /webhooks/stack-auth` endpoint (NO auth dependency — webhooks come from Stack Auth, not authenticated users)
    - Accept raw JSON body
    - Handle event types:

      `user.created`:
      1. Extract `user_id` (Stack Auth ID) and `email` from event payload
      2. Check if a user with this email already exists in the `users` table (for migration: match existing users by email)
      3. If exists: UPDATE the existing row, SET `stack_auth_id = event.user_id`
      4. If not exists: INSERT new row with a generated UUID as `id`, `stack_auth_id = event.user_id`, `email = event.email`, `role = 'customer'` (default), `full_name = event.display_name` or empty
      5. This handles AUTH-09: existing users matched by email get linked; new users get created

      `user.updated`:
      1. Extract `user_id` and metadata from event payload
      2. If `clientReadOnlyMetadata.role` is present, UPDATE `role` in users table WHERE `stack_auth_id = user_id`
      3. Also update `email`, `full_name` if changed

    - For webhook security: Check `x-stack-webhook-secret` header against a configured secret (add `STACK_WEBHOOK_SECRET` to backend config.py). If mismatch, return 401. If the secret isn't configured, log a warning but still process (for development).

    **Part D: Register webhook router in main.py**
    - Import `webhooks_router` from `app.api.v1.webhooks`
    - Add `app.include_router(webhooks_router, prefix=settings.API_V1_PREFIX)`
