  <action>
    **Step 1: Find every file that imports a legacy auth hook.**
    Search the entire `frontend/src/` directory:
    ```
    grep -rl "useAuth\|useSimpleAuth\|useAuth_replit" frontend/src/ --include="*.tsx" --include="*.ts"
    ```
    Also check `frontend/src/components/owner/` subdirectory for any owner-specific components using auth.

    **Step 2: For each file found, apply the migration pattern:**

    REMOVE any of these imports:
    ```typescript
    import { useAuth } from '@/hooks/useAuth'
    import { useAuth } from '@/hooks/useSimpleAuth'
    import { useAuth } from '@/hooks/useAuth_replit'
    ```

    ADD:
    ```typescript
    import { useUser } from "@stackframe/stack";
    ```

    **Step 3: Update hook usage patterns in each component.**

    Old patterns → New patterns:

    a) Auth state:
    - `const { user, isAuthenticated, isLoading } = useAuth()` → `const user = useUser();`
    - `isAuthenticated` checks → `!!user`
    - `isLoading` → Stack Auth handles loading internally (if you need it: `const user = useUser({ or: "redirect" })` redirects during loading)
    - `if (!isAuthenticated)` guards → `if (!user)`

    b) User data access:
    - `user.email` → `user.primaryEmail`
    - `user.name` or `user.full_name` → `user.displayName`
    - `user.id` → `user.id` (Stack Auth user ID — string, not UUID)
    - `user.role` → `user.clientReadOnlyMetadata?.role || "customer"`

    c) Auth actions:
    - `login(email, password)` → Remove; Stack Auth handler pages handle login
    - `logout()` → `await user.signOut()` (redirects to /handler/sign-in via afterSignOut config)
    - `register(...)` → Remove; Stack Auth handler pages handle registration

    d) Token access:
    - `localStorage.getItem('token')` or `localStorage.getItem('access_token')` → Remove entirely; api.ts interceptor handles token attachment
    - Any direct token passing to API calls → Remove; the interceptor handles it

    e) Protected page pattern:
    - `if (!isAuthenticated) return <Redirect to="/login" />` → `const user = useUser({ or: "redirect" })` which auto-redirects unauthenticated users to sign-in

    f) Role-gated rendering:
    - `if (user.role === "owner")` → `if (user?.clientReadOnlyMetadata?.role === "owner")`
    - Use optional chaining since clientReadOnlyMetadata may be null for new users

    **Step 4: Update `page.tsx` (main SPA page):**
    - This file uses conditional rendering based on auth state for different "pages" (home, explore, dashboard, etc.)
    - Replace auth hook usage with `useUser()`
    - For protected views (dashboard, owner-dashboard): check `!!user` and role before rendering
    - For redirect-to-login on protected views: use `useUser({ or: "redirect" })` or manual redirect to `/handler/sign-in`

    **Step 5: Check owner/ subdirectory components.**
    `frontend/src/components/owner/` may contain components that access tokens directly. Search and update them too.
  </action>
