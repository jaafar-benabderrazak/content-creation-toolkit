<task type="auto">
  <name>Task 2: Update Navbar + API client for Stack Auth</name>
  <files>
    librework/frontend/src/components/Navbar.tsx
    librework/frontend/src/lib/api.ts
  </files>
  <action>
    Rewrite auth-related parts of `src/components/Navbar.tsx`:
    - REMOVE: any import of `useAuth`, `useSimpleAuth`, or `useAuth_replit`
    - ADD: `import { useUser } from "@stackframe/stack";`
    - Replace all auth state logic with: `const user = useUser();`
    - `user` is null when not signed in, an object when signed in
    - Replace `isAuthenticated` checks with `!!user`
    - For user display: `user.displayName` or `user.primaryEmail`
    - For role checks (showing owner menu items): `user?.clientReadOnlyMetadata?.role === "owner"` or `=== "admin"`
    - For sign-out: `user.signOut()` — per user decision, this should redirect to login page (Stack Auth handles redirect via the `afterSignOut` URL in client config)
    - For sign-in link: navigate to `/handler/sign-in`
    - For sign-up link: navigate to `/handler/sign-up`
    - Keep all non-auth Navbar logic (navigation links, mobile menu, styling) unchanged

    Rewrite auth interceptor in `src/lib/api.ts`:
    - REMOVE: the request interceptor that reads from `localStorage.getItem('access_token')` or `localStorage.getItem('token')`
    - REMOVE: the response interceptor that handles 401 by calling refresh token endpoint and retrying
    - REMOVE: any `localStorage.removeItem` / `window.location.href = '/login'` logout logic
    - ADD: new request interceptor that:
      1. Imports `stackClientApp` from `@/stack/client`
      2. Gets the current user: `const user = await stackClientApp.getUser();`
      3. If user exists: `const { accessToken } = await user.getAuthJson();`
      4. Sets header: `config.headers["x-stack-access-token"] = accessToken;`
      5. Returns config
    - Keep the axios instance creation (`axios.create({ baseURL: ... })`) unchanged
    - Keep the `export default api` unchanged

    IMPORTANT: The axios interceptor `use()` callback becomes async. Axios supports async interceptors — this is fine.
  </action>
  <verify>
    1. `grep "useUser" src/components/Navbar.tsx` returns matches
    2. `grep "useAuth\|useSimpleAuth\|useAuth_replit" src/components/Navbar.tsx` returns NO matches
    3. `grep "x-stack-access-token" src/lib/api.ts` returns a match
    4. `grep "localStorage" src/lib/api.ts` returns NO matches
    5. `grep "refresh" src/lib/api.ts` returns NO matches (old refresh interceptor removed)
  </verify>
  <done>Navbar uses useUser() from Stack Auth — shows auth state correctly, role-based menu items, sign-out works. API client sends x-stack-access-token header via Stack Auth SDK, no localStorage token management.</done>
