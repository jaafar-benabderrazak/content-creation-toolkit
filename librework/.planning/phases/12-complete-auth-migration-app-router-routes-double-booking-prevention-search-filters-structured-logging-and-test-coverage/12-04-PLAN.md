  <action>
    Create `backend/scripts/migrate_users_to_stack_auth.py` — a standalone Python script that:

    1. Reads all users from the Supabase `users` table (via Supabase client).
    2. For each user where `stack_auth_id` is NULL:
       a. Calls Stack Auth's server-side management API to create or find a user by email.
       b. If user exists in Stack Auth (matched by email), retrieves their Stack Auth user ID.
       c. If user does not exist in Stack Auth, creates them (email only — password must be reset on first login since legacy password hashes cannot be imported).
       d. Updates the Supabase `users` table: `SET stack_auth_id = '{stack_auth_user_id}' WHERE id = '{user_id}'`.
    3. Logs progress: number of users processed, migrated, skipped (already has stack_auth_id), failed.
    4. Uses environment variables: `SUPABASE_URL`, `SUPABASE_SERVICE_KEY`, `STACK_PROJECT_ID`, `STACK_SECRET_SERVER_KEY`.

    The script is idempotent — running it twice does not create duplicate Stack Auth users (checks by email first).

    Include a `if __name__ == "__main__"` block and a `--dry-run` flag that prints what would be done without making changes.

    Note: This script is a placeholder for production use. In dev/staging, users can simply re-register. The script handles the production case where real user accounts exist.
